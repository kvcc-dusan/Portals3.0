# CMS AI Alt-Text Generation Demo

This is a standalone demo for the AI alt-text generation feature, designed as a clean, handoff-ready example for backend developers integrating this logic into the existing CMS (Portals).

## Structure
- `backend/` — Node.js + Express server handling the Gemini API calls.
- `app/` (Frontend) — React + Vite + Tailwind app simulating the CMS image field.

---

## How to Run Locally

### 1. Backend Setup
The backend requires a Google Gemini API key.

1. Open `backend/.env` and ensure your `GEMINI_API_KEY` is set.
2. Install dependencies and run the server:
   ```bash
   cd backend
   npm install
   npm run dev
   ```
   The backend will start on `http://localhost:3001`.

### 2. Frontend Setup
1. Open a new terminal window.
2. Install dependencies and start the Vite dev server:
   ```bash
   cd app
   npm install
   npm run dev
   ```
3. Open the provided `localhost` URL (usually `http://localhost:5173` or `5174`) in your browser.

---

## How it Works

1. **Frontend**: The `MediaCard` component in `app/src/AltTextTest.tsx` accepts an image via drag-and-drop or file selection.
2. Clicking **Generate** converts the image to a base64 string.
3. It sends a `POST` request to the backend with the base64 string and MIME type.
4. **Backend**: Receives the request and uses the `@google/generative-ai` SDK (`gemini-2.0-flash` model) to generate an alt-text description.
5. The result is sent back to the frontend and displayed in the alt-text field.

---

## API Endpoint Details (For Backend Integration)

**`POST /generate-alt-text`**

**Request (`application/json`)**
```json
{
  "image": "<base64_string_without_data_prefix>",
  "mimeType": "image/jpeg",
  "systemPrompt": "Optional custom instructions..."
}
```

**Response (`application/json`)**
```json
{
  "altText": "A descriptive alt text generated by Gemini."
}
```

### ⚠️ Note for Production
In this demo, we send the raw base64 image data from the frontend to the backend. **In production**, if the image is already uploaded and hosted via a URL or stored on cloud storage (like S3), you should **pass the image URL or the file path** instead of the base64 string, and adjust the Gemini API call on the backend to fetch the image directly. This saves significant bandwidth between the frontend and backend.
